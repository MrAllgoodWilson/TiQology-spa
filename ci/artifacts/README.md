# CI Artifacts Directory

This directory stores the output of database check runs.

## Contents

Artifacts are generated with timestamps and include:

- `rls_checks_*.txt` - Row Level Security policy validation results
- `index_checks_*.txt` - Index health and usage analysis
- `table_stats_*.txt` - Table statistics and bloat information
- `performance_tests_*.txt` - Query performance test results
- `benchmark_queries_*.txt` - Database benchmark results
- `query_performance_*.txt` - Combined query performance analysis
- `rls_validation_*.txt` - Specific RLS validation reports

## Usage

These artifacts are automatically generated by the GitHub Actions workflow `db_checks.yml` and can also be generated locally by running the scripts in `ci/scripts/`.

## Retention

Artifacts may be retained according to GitHub Actions retention policies or can be manually cleaned up as needed.
